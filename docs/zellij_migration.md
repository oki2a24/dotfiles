# Zellij 移行作業の記録

このドキュメントは、`tmux` から `zellij` への移行作業に関する記録です。

## 概要

`tmux` の代替として `zellij` を導入し、設定を dotfiles で管理することを目指しています。

## 作業の進捗

### 完了した作業

-   **`zellij` の導入**:
    -   `brew/bundle/mac/Brewfile` に `zellij` を追加。
    -   `link.sh` に `zellij/config.kdl` へのシンボリックリンク作成処理を追加。
    -   `zellij/config.kdl` を生成し、基本的な設定（マウス操作、クリップボード連携、セッション復元、`onedark` テーマの適用、`tmux` ライクなキーバインドの一部）を適用済み。

### 一時的に元に戻した変更

-   `zjstatus` プラグイン関連の変更は一時的に元に戻しています。プラグインの `.wasm` ファイルはリポジトリから削除され、`link.sh` および `zellij/config.kdl` の設定もデフォルト状態に戻されています。

## 残された課題（次回のタスク）

`zellij` の初期設定に関して、以下の課題が残っています。

1.  **ステータスバーに日付と時刻が表示されない**:
    *   `zjstatus` プラグインの導入を試みましたが、権限の問題やプラグインロードの失敗により表示されていません。
    *   次回は、`zjstatus` を再度導入し、ロードできない原因を特定します。
2.  **マウスでのペインリサイズができない**:
    *   `mouse_mode true` に設定済みですが、マウスでの直感的なリサイズが機能していません。`zellij` の仕様または設定の問題を調査します。
3.  **`vi`ライクなコピーモード (`v`, `y` で選択・コピー) の設定**:
    *   `scroll` モードでの `StartSelection` アクションの記述が原因で `zellij` が起動しなくなったため、現在は無効化されています。
    *   `vi` ライクなコピーモードを実現するための正しい設定方法を再調査・適用します。
4.  **新規タブ作成時にカレントディレクトリを引き継ぐ (`Ctrl-b c`)**:
    *   `Run "sh" -c "zellij action new-tab -c . &"` を試しましたが、タブが追加されず右側にペインが出現する問題が発生したため、一旦デフォルトの動作に戻しています。
    *   次回、この問題の解決方法を再検討します。
